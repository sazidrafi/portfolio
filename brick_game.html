— Overview

Project Description

A Brick Breaker game built with pure JavaScript and Canvas, featuring adaptive difficulty. The game includes three levels (Easy, Medium, Hard), where ball and paddle speed scales with performance, and Hard mode introduces score penalties for missed balls.

This project demonstrates advanced game mechanics, physics simulation, and dynamic difficulty adjustment based on player performance. Built entirely with vanilla JavaScript without external game engines or libraries.

— Features

Key Features

Multi-Level Difficulty

Three difficulty modes (Easy, Medium, Hard) with unique characteristics and progressive challenges.

Speed Scaling

Ball and paddle speed increase dynamically with score to maintain challenge throughout gameplay.

Score System

Earn +10 points per brick, with -10 penalty on Hard mode for missed balls.

Responsive Controls

Smooth paddle movement using keyboard arrow keys with precision control.

UI Feedback

Real-time score penalty messages and speed statistics display during gameplay.

Level Progression

Automatic progression to next level or victory screen upon completion.

— Mechanics

Game Mechanics

Physics & Collision Detection

  • Ball Physics: Realistic ball bouncing with proper angle calculations and velocity management
  • Wall Collisions: Accurate detection and reflection off canvas boundaries
  • Paddle Interaction: Dynamic bounce angles based on hit position for strategic gameplay
  • Brick Collision: Efficient detection algorithm for brick destruction with immediate visual feedback

Adaptive Difficulty System

Dynamic Speed Scaling: As players break more bricks and score increases, both ball speed and paddle speed automatically increase. This creates a progressively challenging experience that keeps players engaged and prevents the game from becoming too easy or repetitive.

Scoring System

  • +10 points for each brick destroyed
  • -10 points penalty in Hard mode when ball is missed
  • Real-time score display with visual updates
  • Speed statistics showing current ball and paddle velocities
  • High score tracking across gameplay sessions

Lives System

  • Start with 3 lives in all difficulty modes
  • Lose a life when ball falls below paddle
  • Game over when all lives are depleted
  • Visual lives counter display with icons

— Challenges

Technical Challenges & Solutions

Challenge: Balancing Difficulty

Solution: Tweaked paddle width and speed multipliers iteratively through extensive playtesting. Created a formula that scales difficulty based on brick count and level progression, ensuring each level feels distinct yet fair.

Challenge: Smooth Ball-Paddle Collisions

Solution: Added directional bounce logic (dy = -dy) and implemented angle variation based on where the ball hits the paddle. Hit position on paddle edge creates sharper angles, while center hits produce straighter trajectories.

Challenge: Progressive Speed Increase

Solution: Implemented a dynamic speed multiplier system that increases incrementally with score. Capped maximum speed to prevent unplayable scenarios while maintaining challenge. Speed display keeps players informed of current difficulty.

Challenge: Brick Collision Accuracy

Solution: Used rectangular collision detection with proper boundary checking. Implemented immediate direction reversal upon collision to prevent ball getting stuck inside bricks. Added visual feedback for brick destruction.

— Implementation

Technical Implementation

Core Technologies

  • Canvas API: For rendering game graphics, animations, and real-time updates
  • RequestAnimationFrame: For smooth 60 FPS gameplay loop and consistent frame timing
  • Event Listeners: For responsive keyboard input handling and real-time controls
  • CSS Animations: For UI feedback messages and smooth transitions

Game Loop Architecture

  • Update game state (ball position, paddle movement, collisions, score)
  • Check win/lose conditions and level completion
  • Render all game objects (paddle, ball, bricks, UI elements)
  • Handle user input for paddle movement with smooth transitions
  • Apply physics calculations and dynamic speed adjustments

No External Libraries

This game is built entirely with vanilla JavaScript, HTML5 Canvas, and CSS - no external game engines or libraries. This demonstrates strong understanding of core web technologies, game development fundamentals, and efficient code organization.
